# 已更新记录

## 2026-04-01

- 火车票日期解析增强（命令与 LLM 工具同时生效）：
  - `train_utils.py` 新增对 `YYYY年M月D日`、`M月D日`、`M月D号`、`M-D`、`M/D` 的解析支持
  - 新增 `大后天` 相对日期支持
  - 无年份的月日日期默认按“最近可用日期”推断：若当年该日期已过，则自动滚到下一年
  - 日期前置命令也可识别新格式（如 `/火车票 4月20号 厦门 上海`）
- `main.py` 的两个 `train_query` 入口均新增可选参数 `travel_date`，可从自然语言中透传日期到查询链路
- `python -m py_compile main.py train_utils.py` 已通过
- 记忆钩子优化（性能 + 提示词预算）：
  - `memory_utils.py` 的 `observe_user` 增加写入节流，支持配置 `memory_observe_user_throttle_seconds`，在节流窗口内跳过重复写库（昵称变化仍会立即写入）
  - `memory_utils.py` 的 `search_related_memories` 增加“相关人物检索缓存索引”，缓存人物条目及候选词权重，查询时不再每次全表重建
  - 缓存索引进一步优化为“倒排索引 + 前缀扫描匹配”：先在消息里提取命中候选词，再按倒排映射聚合用户分数，减少逐用户逐候选判断
  - 新增相关日志观测：索引重建会记录 entries/candidates/mappings/cost_ms，检索阶段会记录 cache_hit、命中候选数和命中用户数（debug）
  - 新增 `memory_related_memory_cache_ttl_seconds`（默认 60 秒），可控制相关人物索引缓存有效期；记忆写操作会自动失效缓存
  - `main.py` 的 `on_llm_request` 增加记忆注入预算器，支持配置 `memory_prompt_max_chars`，按分段配额裁剪注入内容，避免 system prompt 膨胀
  - 人物联动回忆改为按需触发：支持 `memory_reminiscence_enabled`、`memory_reminiscence_min_message_length`，仅在命中回忆线索或提及相关人物时触发联动检索
  - `_conf_schema.json` 新增上述 5 个 memory 配置项
  - `python -m py_compile main.py memory_utils.py passive_memory_utils.py` 与 `_conf_schema.json` JSON 校验已通过
